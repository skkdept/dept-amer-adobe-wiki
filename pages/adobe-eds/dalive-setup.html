da.live Setup

Get started with Adobe Edge Delivery Services using the da.live platform. Learn how to set up your first project, configure content delivery, and optimize performance.

What is da.live?

da.live (Digital Assets live) is Adobe's edge delivery content distribution platform built for exceptional performance. It enables rapid content delivery with built-in performance optimization, instant global distribution, and seamless integration with Adobe's creative tools.

Prerequisites

Before starting with da.live, ensure you have:

  • Adobe Account - Active Adobe Creative Cloud or Enterprise subscription
  • GitHub Account - For repository management and version control
  • Node.js - Latest LTS version installed locally
  • Git CLI - For cloning and managing repositories
  • Text Editor - VS Code or similar for code editing

Project Initialization

Get started with da.live using the quickstart template:

Step 1: Create GitHub Repository

Start by creating a new repository on GitHub or clone the Adobe da.live template:

git clone https://github.com/adobe/da-live-quickstart.git my-project cd my-project

Step 2: Install AEM CLI

Install the AEM CLI tool for local development:

npm install -g @adobe/aem-cli

Step 3: Install Dependencies

Install all project dependencies:

npm install

Step 4: Configure Environment

Create a .env file with your da.live configuration:

DA_LIVE_DOMAIN=my-site.da.live REPO_NAME=my-repo REPO_OWNER=my-github-username GITHUB_TOKEN=your_github_token

Local Development

Set up your local development environment:

Start Development Server

Run the local AEM development server:

aem up

This starts a local server at http://localhost:3000 with hot reloading enabled.

Content Structure

Organize your content in Google Sheets or MS Excel as the content source:

  • Shared Drive - Authorized Google Drive or SharePoint location
  • Sheet Structure - Tables define content models and data
  • Metadata - Include SEO, publishing, and visibility metadata

Content Management

Manage content through multiple methods:

Spreadsheet-Based Publishing

Content is authored in Google Sheets or Excel and automatically synced to da.live:

# Push content updates aem sync

Direct File Editing

Create and edit .md (Markdown) files directly in your repository

Bulk Publishing

Publish multiple pages at once using the AEM CLI:

aem publish --folder /content/pages --recursive

Performance Optimization

da.live automatically optimizes content for edge delivery, but follow these best practices:

  • Image Optimization - Use WebP and AVIF formats with fallbacks
  • Lazy Loading - Defer off-screen image loading
  • CSS/JS Bundling - Minimize and bundle assets
  • Caching Strategy - Configure edge cache headers appropriately
  • Content Compression - Enable gzip and brotli compression

Analytics & Monitoring

Track performance and user behavior:

Performance Metrics

  • Core Web Vitals - LCP, FID, CLS metrics from Google
  • Edge Performance - Request/response times from edge nodes
  • Cache Hit Ratio - Percentage of requests served from cache

Deployment Process

Deploy your project to production:

Production Build

npm run build

Push to main Branch

Commits to your main branch automatically trigger production deployment:

git add . git commit -m "Production update" git push origin main

Verify Deployment

Check deployment status in the GitHub UI or using the AEM CLI:

aem status

Essential da.live Commands

  • aem up - Start local development server
  • aem sync - Sync content from Google Sheets
  • aem publish - Publish content to production
  • aem status - Check deployment and sync status
  • npm run build - Build for production
