Universal Editor Setup

Master Adobe's Universal Editor for seamless content creation and visual editing. Enable non-technical teams to manage content with powerful authoring capabilities.

What is Universal Editor?

The Universal Editor is Adobe's next-generation editing platform that provides a unified, visual content editing experience. It's designed to work across different content sources and enables front-end developers and content authors to collaborate seamlessly.

Key Features

  • Visual Editing - WYSIWYG interface for content management
  • Flexible Content Models - Support for custom content structures
  • Real-time Preview - See changes instantly across devices
  • Collaboration - Multiple authors can work simultaneously
  • Version Control - Track changes and revert when needed
  • Extensibility - Customize with custom components and plugins

Prerequisites

Ensure you have the following set up:

  • AEM Instance - Running AEM as a Cloud Service or on-premises
  • Universal Editor Service - Deployed and accessible
  • Adobe IMS Access - Valid Adobe organization
  • Author Permissions - Content authoring rights
  • Browser Support - Chrome, Firefox, Safari, or Edge (latest versions)

Installation & Configuration

Step 1: Enable Universal Editor

Configure Universal Editor in your AEM Cloud Service instance:

# Update pom.xml to include Universal Editor dependencies <dependency> <groupId>com.adobe.aem</groupId> <artifactId>aem-universal-editor-core</artifactId> <version>1.x.x</version> </dependency>

Step 2: Configure Content Source

Specify where your content lives (Google Sheets, AEM, or custom backend):

{ "contentSource": "https://your-content-api.com/api", "authToken": "your_auth_token", "model": "page", "fields": [ {"name": "title", "type": "string"}, {"name": "description", "type": "text"}, {"name": "image", "type": "image"} ] }

Step 3: Create Editable Components

Define which components are editable in the Universal Editor:

// components/text/text.js export default { name: 'Text Block', label: 'Text Block', fields: [ { name: 'content', type: 'richtext', label: 'Content' } ] }

Setting Up the Editor Interface

Configure Authoring Experience

Customize the visual editing interface for your team:

  • Property Panel - Configure fields shown to authors
  • Component Library - Organize available components
  • Presets - Save common configurations
  • Workflows - Define publish and approval flows

Create Content Models

Define the structure of your content:

const pageModel = { title: { type: 'string', required: true, maxLength: 100 }, slug: { type: 'string', pattern: '^[a-z0-9-]+$' }, content: { type: 'array', items: { type: 'object', properties: { type: 'string', content: 'string' } } }, seoMetadata: { type: 'object', properties: { description: 'string', keywords: 'array' } } }

User & Permission Management

Control who can edit and publish content:

User Roles

  • Admin - Full system access and configuration
  • Editor - Create and edit content
  • Reviewer - Approve content before publishing
  • Publisher - Publish approved content
  • Viewer - Read-only access

Permission Configuration

# Grant editor permissions to a user aem users grant-role \ --user user@example.com \ --role editor \ --resource /content/pages/*

Publishing & Workflow

Manage your content publication process:

Draft to Publish Workflow

  1. Author creates/edits content in the visual editor
  2. Content is saved as draft automatically
  3. Author requests review of content
  4. Reviewer approves or requests changes
  5. Publisher publishes approved content
  6. Content is deployed to Edge Delivery Services

Publish Changes

# Programmatically publish content curl -X POST https://your-instance.aem.cloud/api/publish \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "path": "/content/pages/my-page", "recursive": true }'

Customization

Extend the Universal Editor with custom functionality:

Custom Components

Create custom editable components tailored to your needs:

// components/hero/hero.js export default { name: 'Hero Section', label: 'Hero', fields: [ {name: 'heading', type: 'string', label: 'Heading'}, {name: 'subheading', type: 'text', label: 'Subheading'}, {name: 'cta', type: 'string', label: 'CTA Text'}, {name: 'backgroundImage', type: 'image', label: 'Background'} ] }

Best Practices

  • Clear Field Labels - Use descriptive labels and help text
  • Validation - Prevent invalid content with field validation
  • Preview Devices - Test content on mobile, tablet, and desktop
  • SEO Optimization - Include SEO fields in content models
  • Version History - Regularly back up and archive old versions
  • Performance - Optimize media assets before upload

Troubleshooting

Common issues and solutions:

Content Not Appearing

Ensure your content model matches the expected structure and all required fields are populated.

Publishing Delays

Check AEM Cloud Service job queues and review deployment status in Cloud Manager.
