🏗️ Project Setup: Adobe Commerce (Docker)

This section uses the markshust/docker-magento configuration for a robust local development environment.

1. Installation & Initialization

Run these in your project directory to build the environment and install the Luma/Venia sample data.

Automated Install (Standard for 2.4.8-p3)

curl -s https://raw.githubusercontent.com/markshust/docker-magento/master/lib/onelinesetup | bash -s -- magento.test community 2.4.8-p3

Deploy Sample Data & Sync Schema

bin/magento sampledata:deploy bin/magento setup:upgrade

Reindex the Catalog (Critical for AEM to "see" products)

bin/magento indexer:reindex bin/magento cache:flush

2. Local Dev "Quality of Life" Settings

Disable security features that interfere with local testing.

  • Disable MFA/2FA - bin/magento module:disable Magento_TwoFactorAuth
  • Disable Password Expiration - bin/magento config:set admin/security/password_is_forced 0

Admin Credentials

  • URL - https://magento.test/admin
  • User - john.smith
  • Pass - password123

🔗 AEM to Magento Handshake (OSGi)

AEM must be configured to trust your local SSL certificate and use the correct "pipe" identifier. Navigate to your OSGi ConfigMgr.

1. GraphQL Client (~venia)

  • GraphQL Service URL - https://magento.test/graphql (Must be https).
  • GraphQL Service Identifier - venia (Lowercase).
  • Accept self-signed SSL certificates - ✅ Checked.
  • Allow HTTP communication - ✅ Checked.
  • Default HTTP Headers - Authorization : Bearer local-dev.

2. GraphQL Proxy Servlet

  • Commerce GraphQL Endpoint - https://magento.test/graphql.
  • Authentication - None.
  • Removed Headers - Ensure this field is Empty (Delete "Cookie" if it's there).

📂 AEM Cloud Service Mapping

Even with the right URL, AEM won't show products unless the site folder is mapped to your client.

  1. Go to Tools > Cloud Services > CIF Configurations.
  2. Select the Venia folder and click Properties.
  3. GraphQL Client: Select venia (Ensure it's not the default one).
  4. Root Category ID: Set to Mg== (This targets the Luma Product Catalog).
  5. Save & Close.

🎟️ Testing Offers & Promotions

To verify the connection is dynamic, create an offer in Magento and check AEM.

In Magento

Go to Marketing > Promotions > Cart Price Rules.

  • Create Rule - 20% Off Bags.
  • Websites - Main Website.
  • Customer Groups - NOT LOGGED IN.
  • Actions - "Percent of product price discount" | Amount: 20.

In AEM

Open the Products Console. Open the Joust Duffle Bag.

The price components on the page should now show the 20% discount applied automatically.

📊 Quick Terminal "Sanity Check"

If the products don't appear in AEM, run this to see if Magento is actually sending data:

curl -k -X POST -H "Content-Type: application/json" --data '{ "query": "{products(search: \"bag\", pageSize: 1) {items {name}}}" }' https://magento.test/graphql

Success: Returns {"name":"Joust Duffle Bag"}.

Fix: If this works but AEM is blank, the issue is your Identifier (must be venia) or Proxy Config.
